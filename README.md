# subagent-fleet

Run Claude Code-style subagents across your local model fleet.

`subagent-fleet` helps you map each coding subagent to the best local machine and model you own — planner on a small model, implementer on your big Mac mini, reviewer on another node — using **Ollama** and **LiteLLM**.

```text
Your laptop / controller
  Claude Code or custom harness
        ↓
  subagent-fleet generated LiteLLM gateway
        ↓
  ├── M4 Mac mini 64GB  → big coder model
  ├── M4 Mac mini 16GB  → planner / summarizer model
  └── local machine      → fast default model
```

## Why?

Modern coding agents use subagents for planning, implementation, review, testing, and summarization.

But local models usually run like this:

```text
one machine
one model
one endpoint
```

That wastes your local hardware.

`subagent-fleet` lets you run a private local agent swarm:

```text
planner     → small fast model on Mac mini 16GB
implementer → large coder model on Mac mini 64GB
reviewer    → large coder model on Mac mini 64GB
summarizer  → tiny model on laptop
```

The goal is simple:

> Turn your idle Macs, workstations, and local Ollama nodes into a private subagent fleet.

## What this project does

`subagent-fleet` is a config-first tool that generates and manages the glue between:

- **Ollama** model servers
- **LiteLLM** routing/gateway config
- **Claude Code-style subagents**
- Local coding harnesses
- Multi-machine local model setups

It does **not** try to replace Ollama or LiteLLM.

It sits above them and makes local subagent orchestration easy.

## Core idea

You define your fleet once:

```yaml
nodes:
  m5-mac:
    endpoint: http://localhost:11434
    role: controller
    tags: [local, fast]

  m4-mini-64gb:
    endpoint: http://192.168.1.50:11434
    tags: [heavy, coder, reviewer]

  m4-mini-16gb:
    endpoint: http://192.168.1.51:11434
    tags: [small, planner, summarizer]
```

Then define models and map agents to those models:

```yaml
models:
  heavy-coder:
    node: m4-mini-64gb
    ollama_model: qwen2.5-coder:32b
    litellm_alias: claude-sonnet-local
    context: 32768
    timeout: 600

  small-coder:
    node: m4-mini-16gb
    ollama_model: qwen2.5-coder:7b
    litellm_alias: claude-haiku-local
    context: 8192
    timeout: 300

agents:
  planner:
    model: small-coder
    description: Use for planning, file discovery, task decomposition, and summarization.
    tools: [Read, Grep, Glob]

  implementer:
    model: heavy-coder
    description: Use for implementation, bug fixes, refactors, and patch creation.
    tools: [Read, Grep, Glob, Edit, MultiEdit, Bash]

  reviewer:
    model: heavy-coder
    description: Use after implementation to review diffs, tests, regressions, and maintainability.
    tools: [Read, Grep, Glob, Bash]
```

`subagent-fleet` generates:

```text
litellm_config.yaml
.claude/agents/planner.md
.claude/agents/implementer.md
.claude/agents/reviewer.md
.env.subagent-fleet
```

## Use cases

- Run Claude Code-style subagents across multiple local Macs
- Use a large local model only for expensive coding tasks
- Use small models for planning, summarization, and memory
- Route subagents by role instead of sending everything to one model
- Keep coding private and local
- Reduce cloud API usage
- Turn spare machines into useful local AI workers

## Status

MVP CLI implemented.

Install locally for development:

```bash
python -m pip install -e ".[dev]"
```

Quickstart:

```bash
subagent-fleet init
subagent-fleet validate
subagent-fleet discover
subagent-fleet generate
subagent-fleet warmup
subagent-fleet status
```

The `discover`, `status`, and `warmup` commands contact the Ollama endpoints listed in `fleet.yaml`. Offline or unreachable nodes are reported without crashing the whole command.

## CLI

```bash
subagent-fleet init
```

Creates:

```text
fleet.yaml
```

```bash
subagent-fleet discover
```

Finds Ollama nodes and models.

```bash
subagent-fleet validate
```

Validates `fleet.yaml`.

```bash
subagent-fleet generate
```

Generates LiteLLM and Claude Code subagent files.

```bash
subagent-fleet warmup
```

Preloads configured models with `keep_alive`.

```bash
subagent-fleet status
```

Shows node health, loaded models, latency, and routing.

```bash
subagent-fleet doctor
```

Shows config validity, local-network security guidance, and Ollama worker setup hints.

```bash
subagent-fleet clean
```

Lists or removes generated files.

## Generated LiteLLM config

`subagent-fleet` can generate a LiteLLM config like this:

```yaml
model_list:
  - model_name: claude-sonnet-local
    litellm_params:
      model: ollama_chat/qwen2.5-coder:32b
      api_base: http://192.168.1.50:11434
      api_key: ollama
      timeout: 600
    model_info:
      max_input_tokens: 32768

  - model_name: claude-haiku-local
    litellm_params:
      model: ollama_chat/qwen2.5-coder:7b
      api_base: http://192.168.1.51:11434
      api_key: ollama
      timeout: 300
    model_info:
      max_input_tokens: 8192

litellm_settings:
  drop_params: true
  master_key: os.environ/LITELLM_MASTER_KEY

router_settings:
  routing_strategy: simple-shuffle
  num_retries: 1
  timeout: 600
```

Start LiteLLM:

```bash
export LITELLM_MASTER_KEY="sk-local-dev"

litellm \
  --config ./litellm_config.yaml \
  --host 0.0.0.0 \
  --port 4000
```

## Using with Claude Code

Point Claude Code at the LiteLLM gateway:

```bash
export ANTHROPIC_BASE_URL="http://localhost:4000"
export ANTHROPIC_AUTH_TOKEN="sk-local-dev"

export ANTHROPIC_DEFAULT_SONNET_MODEL="claude-sonnet-local"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="claude-haiku-local"

claude
```

Now your subagents can route to different local Ollama machines through LiteLLM.

## Security model

By default, this project is designed for private local networks.

Recommended setup:

```text
LAN only
or
Tailscale / WireGuard only
```

Avoid exposing Ollama or LiteLLM directly to the public internet.

Recommended:

```bash
OLLAMA_HOST=0.0.0.0:11434
```

Only when the machine is protected by:

- local trusted network
- firewall rules
- Tailscale
- WireGuard
- private subnet

## What this is not

`subagent-fleet` is not:

- a new inference engine
- a replacement for Ollama
- a replacement for LiteLLM
- a model sharding framework
- Kubernetes for local LLMs

It is a workflow layer for local agent orchestration.

## What this is

`subagent-fleet` is:

- config-first
- local-first
- subagent-aware
- Ollama-compatible
- LiteLLM-powered
- Claude Code-friendly
- useful for multi-machine developer setups

## Why this matters

Local AI coding is moving from:

```text
one model answers everything
```

to:

```text
many specialized agents collaborate
```

But local infrastructure has not caught up.

Developers already have:

- MacBooks
- Mac minis
- gaming PCs
- home servers
- spare GPUs
- Ollama models
- coding harnesses

`subagent-fleet` connects them into one local developer workflow.

## Project philosophy

- Use existing tools.
- Do not reinvent inference.
- Do not hide the config.
- Make local hardware useful.
- Prefer simple routing over complex orchestration.
- Keep everything private by default.
- Make subagent execution visible.
